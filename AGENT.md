# AGENT.md: Standard Operating Procedures (SOP)

## Project Context: LLMIndex
LLMIndex is a unified LLM registry designed for transparency, portability, and Git-Ops automation. It merges OpenRouter (metadata) and ArtificialAnalysis (benchmarks) into a portable SQLite/JSON format.

## Foundational Architecture
This project MUST follow **Clean Architecture** principles to separate business logic from technical infrastructure:

- **Layer 1: Domain**: Entities (`LLMModel`), Value Objects, and abstract Repository/Service interfaces. NO dependencies on frameworks or libraries.
- **Layer 2: Use Cases**: Application-specific business rules (e.g., `SyncRegistryUseCase`, `MatchModelsUseCase`).
- **Layer 3: Adapters**: 
    - **Controllers**: FastAPI route handlers.
    - **Persistence**: SQLite and JSON implementations of repository interfaces.
    - **Gateways**: External API fetchers (OpenRouter, ArtificialAnalysis).
- **Layer 4: Infrastructure**: Dependency injection, environment config (`.env`), and GitHub Actions workflows.

## Mandatory Skills & Patterns
All development must adhere to the patterns defined in:
1. `architecture-patterns`: Strict Clean Architecture layering.
2. `async-python-patterns`: Non-blocking I/O using `asyncio` and `aiohttp`.
3. `fastapi-templates`: Schema-first development with Pydantic and standard DI.
4. `python-testing-patterns`: Comprehensive unit/integration tests with `pytest` and mocks.
5. `workflow-orchestration-patterns`: Deterministic logic for the data pipeline.

## Development Rules
- **Schema First**: Use `docs/02-data-schema.md` as the master source for data models.
- **Git-Ops Ready**: All pipeline outputs must be deterministic. Avoid random numbers or non-reproducible shifts in the registry.
- **No Heavy Infra**: Strictly keep SQLite as the engine and JSON as the delivery format.
- **Documentation Over Code**: Update the modular `docs/` before implementing any significant change in logic.
- **Secret Handling**: Never hardcode API keys. Use environment variables and refer to local `.env` vs production GitHub Secrets.
