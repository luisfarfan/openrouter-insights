# AI Context: Navigation & Development Rules

## For AI Agents & Developers
This document provides the context required for high-fidelity code generation and architectural navigation within the **LLMIndex** project.

### Project Philosophy
- **Lite & Portable**: No Postgres, no complex infra. Everything in SQLite/JSON.
- **Spec Driven**: The `docs/` folder is the source of truth. Codes must align with specs.
- **Modular Backend**: Services are separated by responsibility (Fetcher, Matcher, Classifier, Exporter).

### Key Rules
1. **Schema First**: Use the `LLMModel` schema defined in `02-data-schema.md` for all data transformations.
2. **Matching Engine**: Refer to `03-matching-engine.md` for normalization rules before attempting any merging logic.
3. **Open Source Friendly**: Ensure all code is documented and easy to set up with a simple `pip install` or `docker build`.
4. **Git-Ops Ready**: All storage operations must be designed to be deterministic (no random data) so that GitHub Actions can run-sync-commit cleanly.
5. **Secret Handling**: Read all API keys from **environment variables** only. For local, use `.env`; for production, use **GitHub Secrets**.
